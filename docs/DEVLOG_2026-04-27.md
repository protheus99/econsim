## 1. Corporate Goal System (CorporateRoadmap)

### New file: `js/core/corporations/CorporateRoadmap.js`

Replaced the old `BoardMeeting` class with a three-horizon planning system:

- **SHORT** (0‚Äì3 months): Execute immediately
- **MEDIUM** (3‚Äì12 months): Queue until prerequisites met
- **LONG** (12+ months): Strategic intent, decomposes into MEDIUM

Key features:
- `seed()` ‚Äî initialises a corporation's roadmap at creation based on tier
- `update()` ‚Äî runs monthly: promotes actions, evaluates goals, returns ready actions
- Promotion conditions are tier-gated (e.g. SEMI_RAW actions unlock when RAW firms exist)
- `DOWNSTREAM_MAP` / `UPSTREAM_MAP` ‚Äî static persona-to-persona vertical integration tables
- `_considerVerticalIntegration()` ‚Äî checks both downstream and upstream paths, prefers acquire over build if a suitable firm is available
- `_checkProfitabilityGoal()` ‚Äî triggers RENEGOTIATE / REDUCE_HEADCOUNT / CLOSE_FIRM based on consecutive loss months
- Full serialization via `getSerializableState()` / `restoreFromJSON()`

## 6. Bug Fixes

### Circular dependency crash on game create
**Error:** `Cannot access 'INDUSTRY_TIERS' before initialization`

`CorporateRoadmap.js` imported from `Corporation.js`, which imported from `CorporateRoadmap.js`. Also `BoardMeeting.js` imported from `Corporation.js` creating a three-way cycle.

**Fix:** Inlined `INDUSTRY_TIERS` and `GOAL_TYPES` as local constants in both `CorporateRoadmap.js` and `BoardMeeting.js`. No more cross-imports into `Corporation.js` from those files.

---

### Mineral coverage corporations missing from corporation list

**Symptom:** Firm showed `corporationAbbreviation: 'AEN'` but no corporation "AEN" appeared in corporations.html.

**Root cause:** In `SimulationEngine.js`:

```javascript
this.corporations = this.corporationManager.generateCorporations();
```

`ensureMineralCoverage()` (called inside `generateCorporations`) pushes mineral corps to `this.engine.corporations` ‚Äî which at that moment is the original array. The assignment then replaces the reference, orphaning those corps.

**Fix:** Changed to push rather than replace:

```javascript
const newCorps = this.corporationManager.generateCorporations();
this.corporations.push(...newCorps);
```

Mineral coverage corps (pushed directly to `this.corporations` via `this.engine.corporations.push`) are now retained in the same array.
